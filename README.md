# Сравнительный анализ алгоритмов сортировки

Реализация и сравнение трёх алгоритмов сортировки с замером среднего времени
работы на массивах разного размера и проверкой соответствия результатов
теоретической сложности O(n log n). Вся работа выполнена в Google Colab.

[![Open In Colab](https://colab.research.google.com/assets/colab-badge.svg)](https://colab.research.google.com/github/pavlovm174066-cyber/PavlovSortirovkaColab/blob/main/PavlovSortirovka.ipynb)

## Задание

Реализовать быструю сортировку с медианным опорным элементом, сортировку
расчёской и пирамидальную сортировку. Для каждого из размеров массива
(2 000, 5 000, 10 000, 22 000, 53 000, 100 000, 220 000, 330 000, 510 000,
730 000 элементов) сгенерировать по 3 случайных массива с фиксацией зерна генератора, запустить на них все
сортировки и вычислить среднее время выполнения каждой. По результатам построить
диаграммы и определить, соответствуют ли они теоретической временной сложности.

## Алгоритмы

- **QuickSort (median-of-3)** — быстрая сортировка с выбором опорного элемента
  по медиане трёх, итеративная реализация (без рекурсии, устойчива на больших массивах).
  Средняя сложность O(n log n).
- **Comb Sort** — сортировка расчёской с фактором сокращения 1.3, память O(1).
- **Heap Sort** — пирамидальная сортировка, гарантированное O(n log n).

Подробное описание алгоритмов, методики и выводы находятся внутри ноутбука.

## Как запустить

Нажмите бейдж **Open In Colab** вверху. В открывшемся ноутбуке выполните
ячейки по очереди сверху вниз. Устанавливать ничего не нужно: библиотека
matplotlib в Colab уже доступна.

1. Запустите три ячейки с сортировками (QuickSort, Comb Sort, Heap Sort) —
   каждая сгенерирует массивы, посчитает среднее время и выведет результаты
   в консоль, а также построит свой график.
2. Скопируйте полученные средние значения из вывода каждой из трёх сортировок
   и вручную вставьте их в списки `quicksort`, `comb`, `heap` внутри четвёртой
   ячейки (сводное сравнение). Значения вводятся в том же порядке размеров
   массива — от 2 000 до 730 000 элементов.
3. Запустите четвёртую ячейку — она построит сводный сравнительный график
   всех трёх сортировок вместе с эталонной кривой O(n log n).

## Методика

Для каждого размера генерируется по 3 массива случайных целых в диапазоне
[0; 1 000 000] с фиксацией `seed`. Время замеряется через `time.perf_counter()` только для самой
сортировки, без операций ввода-вывода, и усредняется по 3 запускам. На графики
наносится эталонная кривая O(n log n), отмасштабированная по последней точке,
для наглядной проверки соответствия.

## Выводы

- **QuickSort** показал наилучший результат на всём диапазоне размеров и рост,
  наиболее близкий к эталону O(n log n): выбор опорного элемента по медиане трёх
  предотвратил деградацию. На максимальном размере (730 000) время составило
  2,08 с — заметно меньше, чем у остальных.
- **Heap Sort** продемонстрировал стабильный рост, соответствующий O(n log n),
  но с бóльшими константами из-за большого числа перестановок и худшей локальности
  обращений к памяти, поэтому уступил быстрой сортировке (5,15 с на 730 000).
- **Comb Sort** оказался конкурентоспособным лишь на малых массивах, однако на
  больших объёмах заметно отклонился вверх от эталонной кривой O(n log n) и
  показал худший результат из трёх (9,01 с на 730 000), что подтверждает его
  практически более высокую сложность на больших n.
- Поскольку все три алгоритма сортировали одни и те же массивы (за счёт фиксации
  зерна генератора `random.seed`), сравнение полностью корректно: различия во
  времени обусловлены исключительно свойствами самих алгоритмов, а не разницей
  во входных данных.
